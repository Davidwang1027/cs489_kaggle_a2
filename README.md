# Task 2
As we assume its following English syntax. First I group words into different groups, which serves as the first layer. The vocab can be divided into groups by syntax, for example, nouns, verbs, adjectives, degree, etc. Then I start doing manual exploration on the binary pair frequency and start building up rules with both the English knowledge and the frequency analysis. I use the `replace_task2.py` to help me with building up rules step by step to see if its reasonable. 

After finalizing the rules, I use the EM using nltk package to train on the probability of rules (`task2.ipynb`). I also add evaluation code to double check the correctness.

# Task 3
Similar from task 2, I also start with manual inspection on the binary pair. I found that sentence length are always even. And seems words comes in pairs, like a b, c d, etc. After grouping them, I also found out that sentences can be group by length, and sentences with same length share very similar structure/vocab. So I do an additional grouping on those pairs, like ab and cd, ef and gh. After two layer of manual work, I struggled to finding out new pattern so I start with the brute force option(inside-outside EM) as implemented in `task3.ipynb`. The idea is basically assume the number of non-terminals N, and construct every single possible rules, as its binary only, there would be N * (N + 9)* (N+9) number of rules. [9 is the number of symbols in the second layer] Then we just do a EM to get the probability.

I tried different number of N from 6-24 (using `task3_sweep.py`) on targeting a 50 preterminal rules at the end. When N = 9 it hits a good balance between the number of rules, parsing success rate and correctness.
